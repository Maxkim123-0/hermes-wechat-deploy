# Brainstorming — 头脑风暴

**把模糊想法变成可执行的方案。**

---

## 流程

```
模糊想法 
    ↓
发散：列举3-5个可能方向（不评判）
    ↓
收敛：选2-3个最有潜力的
    ↓
细化：每个方案的思路、利弊、成本
    ↓
推荐：给出首选建议+理由
```

---

## 输出格式

```
## 方案对比

| | 方案A | 方案B | 方案C |
|:--|:-----|:-----|:-----|
| 思路 | ... | ... | ... |
| 优点 | ... | ... | ... |
| 缺点 | ... | ... | ... |
| 时间 | N天 | N天 | N天 |
| 难度 | ⭐⭐ | ⭐⭐⭐ | ⭐ |

## 推荐：方案X
理由：...
```

---

## 使用场景

- 新产品/功能构思
- 内容选题策划
- 技术方案选型
- 变现路径探索
- 活动创意

---

## 原则

1. **先发散后收敛** — 不要一上来就否定
2. **有依据** — 每个方案说明为什么可行
3. **给选择** — 让用户决策，不替用户决定
4. **落地** — 每个方案给出下一步行动
