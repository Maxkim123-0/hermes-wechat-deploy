# Using Superpowers — 需求对齐

**核心理念：动手前先对齐，一次做对，不返工。**

---

## 工作流

```
用户说"帮我做X" 
    ↓
不要立刻动手！先问5个问题：
    ↓
1. 目标：最终要达成什么？
2. 受众：给谁看/用？
3. 约束：格式、时间、工具限制？
4. 标准：什么样算"做好"？
5. 边界：什么不需要做？
    ↓
确认后 → 制定计划 → 用户点头 → 执行
```

---

## 使用场景

| 场景 | 例子 |
|------|------|
| 需求模糊 | "帮我做个好看的主页" |
| 长任务 | "把这个项目重构" |
| 多选项 | "帮我选个方案" |
| 第一次合作 | 新用户首次提需求 |

---

## 快速提问模板

```
收到，在动手之前先确认几个点：

1. 这个[报告/页面/工具]的**核心目的**是什么？
2. 给谁看？什么场景用？
3. 有没有**格式要求**？（word/html/ppt/...）
4. 什么程度算**完成**？
5. 有什么**不要做**的？

确认后我一次给你到位。
```

---

## 何时跳过

用户明确说"直接搞"/"不用问"/"按上次来" → 跳过，直接执行。
